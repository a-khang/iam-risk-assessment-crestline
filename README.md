# Identity Security: A learning project
This project reflects my learnings of IAM and the role it plays in the cybersecurity strategy for organizations. I focus on these basic components for achieving an IAM Program:
- **Governance**
- **Authentication**
- **Authorization**
- **Accounting**

Under a simulated organizational scenario, I will:
1. <a href="https://github.com/a-khang/identity_security/blob/main/part1.md/">Define why the organization needs IAM, the risks that IAM will address, and who owns what (IT? HR? Managers? Security?)</a>
2. <a href="https://github.com/a-khang/identity_security/blob/main/part2.md/">Establish the access control model, and the considerations for enforcement, exceptions, and change.</a>
3. <a href="https://github.com/a-khang/identity_security/blob/main/part3.md/">Document policies, standards (JML, PAM, JIT) and procedures (Exception management, Access Review) to enforce in the IAM operating model.</a>
4. <a href="https://github.com/a-khang/identity_security/blob/main/part4.md/">Configure Entra ID and demonstrate policy enforcement, control of privileged access, and procedures.</a>

Goals for learning and growth:
- Explaining concepts clearly
- Demonstrate high-level and low-level thinking
- Be able to "solution" (recommend and justify) and how to design a cybersecurity solution in an organizational context

**Future Ideas**
- Tracking incidents involving privilege escalation
- Demo with non-human identities (and Agentic risks)
- Cloud infrastructure entitlement management
